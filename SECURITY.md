# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✓         |

## Reporting a Vulnerability in JSPECTER

If you discover a security vulnerability in JSPECTER itself (not in a target you scanned), please do **not** open a public GitHub issue.

Instead, email the maintainers directly. We will respond within 72 hours and work with you on a coordinated disclosure.

## Scope

This security policy covers vulnerabilities in JSPECTER's own code — for example:
- Scope enforcement bypass (JSPECTER scanning out-of-scope targets)
- Credential leakage in reports or logs
- Dependency vulnerabilities with active exploits

## Ethical Use

JSPECTER is for authorized security research only. Use against systems you do not have explicit written permission to test is prohibited and may violate applicable laws.
